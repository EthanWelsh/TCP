Minet Documentation

WORK IN PROGRESS - this is a suggested skeleton for how to divide up the documentation. There may be a good deal of overlap between the three groups, but the concerns for the three groups are (in my opinion) different enough to warrant splitting up the documentation into three parts.

For Students
A gentle introduction to Minet, the GUI tools that are available, programming/debugging techniques. When I was in 340, I would have also appreciated some tips on how to test that we got the environment up and running correctly -- include tips on how to use nc, as well as tcp_client and tcp_server. (If wireshark is available, a brief guide on how that helps & how to use it). Finally, make students aware of the doxygen-generated documentation.

For Instructors
A technical guide on how to set up Minet in your CS department so that students can use it (including the many options that are available: physical lab environment vs distributing Minet inside VMs). Perhaps include some suggestions for how to use Minet in the curriculum.

For Minet Developers
Describe how to set up a good testing environment (e.g., using the VLAB, or my approach of running two instances of Ubuntu in VirtualBox), especially focusing on how to isolate Minet's network traffic (e.g., by using Host-Only networking). Also, for starters, how to compile Minet, as well as make sense of the makefile, the setup script, etc (but wait on this, since there's a good chance all of this will change). Document which tools are used for which aspects of the project (e.g., bug tracking, documentation, etc), what options need to be changed from the defaults, etc.

Also consider linking to the tech report and the doxygen documentation from here.
